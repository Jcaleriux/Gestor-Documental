Base de datos

La propuesta con PostgreSQL Flexible Server permite respaldos automáticos y restauración a un punto dentro de la ventana de retención configurada.

Storage documental

Los documentos operativos pueden respaldarse como capa separada, evitando que PDFs y XML dependan del nodo de aplicación.

Rollback

El proyecto ya cuenta con runbook de backup y rollback, plan previo de respaldo y criterio para volver atrás si un release falla.

Readiness

Ya existe checklist de despliegue, reporte de preparación, verificaciones básicas y visibilidad de versión y commit del release.

Antes del deployment

Validar y respaldar

Versionado, changelog, validación de preparación, plan de backup, validación de entorno y respaldo de DB y storage.

Deployment

Publicar con evidencia

Aplicar release con secrets reales, rutas correctas y verificaciones básicas mínimas sobre endpoints y flujo base.

Si algo falla

Volver con control

Rollback de código, base de datos y storage según el alcance real del incidente, no por improvisación.

Además del alojamiento de la aplicación, la propuesta contempla respaldo, recuperación y trazabilidad de la versión desplegada.
