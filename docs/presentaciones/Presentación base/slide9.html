Propuesta de despliegue

Arquitectura objetivo en Azure alineada con la base actual del sistema

09 / 15

Principio de diseño

La propuesta no fuerza reescribir la aplicación. Se apoya en el diseño actual: frontend y backend web + PostgreSQL + storage persistente para documentos.

Usuarios internos

Acceso a la aplicación desde red corporativa o rutas controladas.

App Service

Hospeda frontend y backend como servicios administrados para simplificar operación y despliegue.

Azure Database for PostgreSQL Flexible Server

Base de datos administrada compatible con el stack actual y preparada para backups automáticos.

Componentes propuestos

Storage persistente

Azure Files montado para conservar PDFs, XML y adjuntos sin depender del disco efímero de la app.

Key Vault

Secretos y credenciales fuera del código y del repositorio.

Red privada donde aplique

Private Endpoint y controles de red para reducir exposición innecesaria.

Separación clara

Aplicación, base de datos y storage administrados como capas distintas y recuperables.

Azure permite un despliegue administrado, secretos externos, respaldo formal y crecimiento incremental sin cambiar el modelo principal del producto.
Propuesta objetivo, no despliegue ya realizado Azure App Service + PostgreSQL Flexible Server + storage persistente + Key Vault
