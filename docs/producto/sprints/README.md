# Sprints

## Objetivo

Usar este directorio para planificar y cerrar bloques pequenos de trabajo.

## Regla simple

- un sprint no debe ser una lista infinita
- un sprint debe tener objetivo claro
- y cada item debe poder declararse `Done` con criterio real

## Cadencia sugerida

- 1 o 2 semanas

## Contenido recomendado por sprint

- objetivo
- capacidad estimada
- items comprometidos
- riesgos
- resultado real
- aprendizaje
